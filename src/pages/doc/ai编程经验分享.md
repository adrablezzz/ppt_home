# 个人 AI 编程经验分享

> 本文记录我在日常开发中使用 AI 编程工具的实践经验，包括工具选择、使用流程和踩坑总结。

---

## 1. 为什么需要 AI 编程？

传统开发中，大量时间花在：
- **查文档 / 搜索引擎**：找 API 用法、排查报错
- **写样板代码**：CRUD、配置文件、测试用例
- **上下文切换**：从设计到实现反复跳跃

AI 编程工具的价值在于：
- 把"思考如何实现"和"动手写代码"的摩擦降到最低
- 让你聚焦在**需求拆解和代码审查**，而不是语法细节
- 复杂任务中充当"不知疲倦的 pair programmer"

## 2. 主流 AI 编程工具对比

### 两大平台

| | OpenAI | Anthropic |
|---|---|---|
| 定位 | AI 基础设施 | 专家型 AI 工具 |
| 额度 | 较高 | 较低 |
| 国内访问 | 限制少 | 限制多 |
| 主要产品 | ChatGPT、Codex、GPT Image Gen、Sora | Claude、Claude Code |

> **我的选择**：主力用 OpenAI 生态，Claude 作为补充（主要用于写 Agent 规则和文档）。  
> Cursor 也在用，但额度不够时会自动降级到 Codex。

---

### Codex vs Claude Code：角色定位

理解这两个工具的本质区别，能避免用错场景：

#### Claude Code — 决策者（Head）

**适合：** 制定计划、设计架构、编写规则文件、代码审查

**使用体验：**
- 需求不清晰时会**主动追问**，而不是猜测后动手
- 代码收敛性好，产出通常可以直接运行
- 能参与从设计到落地的全流程决策

**适用场景举例：**
- 新项目启动时讨论技术方案
- 写 `AGENT.md`、`SKILL.md` 等规则文件
- 对已有代码做整体 review

---

#### Codex — 执行者（Hands）

**适合：** 执行具体的编码任务

**使用体验：**
- 定位是**快速完成任务**，有时需求还没说清楚就开跑
  - gpt-5.4 这个问题比较明显，gpt-5.5 有改善
- 适合颗粒度小、边界清晰的任务

**解决"乱跑"问题：**  
在项目根目录写 `.codex/AGENT.md`，明确要求 Agent 在开始前主动确认需求边界：

```markdown
# AGENT 行为规范

## 接收任务时
1. 先理解需求，不要立即动手
2. 列出你的理解和假设，向用户确认
3. 明确边界条件（哪些不做、哪些需要兼容）
4. 确认后再开始实现
```

### Codex 实战使用流程

#### 订阅方式（国内用户）

Codex 不支持国内直接订阅，可行方案：
- 用 **PayPal 绑定招商银行信用卡**，再通过 PayPal 完成订阅

#### 客户端选择

- **Windows 客户端**：日常主力，交互最流畅
- **VSCode 插件**：适合边写代码边查询
- **CLI**：适合脚本化流程、CI 场景

#### 我的标准编程流程

复杂任务不要一次性全扔给 Agent，先手动拆分成小任务，按以下循环推进：

```
0. 确认 git 工作区干净（方便 redo）
1. /plan 模式 — 与 Agent 讨论需求，产出计划
2. 审查计划 — 提出修改意见，确认后再执行
3. 审查代码 — 有问题继续修改；方向偏差太大直接 redo
4. 重复 0-3，直到需求完成
```

> **经验：** 每次提交前写好 `TODO.md` 理清需求，是避免 Agent 跑偏的最有效手段。

## 3. OpenAI / Anthropic 生态使用

### OpenAI 生态

**ChatGPT**  
日常问题的首选入口，比搜索引擎更快、答案更聚焦。适合：
- 快速了解一个陌生技术
- 排查报错、解释代码
- 头脑风暴

**GPT Image Gen（gpt-imageGen-2）**  
图片生成质量很高，可以生成高度逼真的图片。直接在 ChatGPT 内使用即可。  
示例：[查看效果](https://chatgpt.com/share/69f054ca-55cc-839e-aa57-55161aa0aa82)

---

### Anthropic 生态

**Claude**  
免费额度有限，我主要用于：
- 编写 / 优化 Agent Skill 文件
- 起草 `AGENT.md` 规则文档
- 对 Codex 生成的代码做二次 review

**Claude Design（claude design）**  
Claude 的 UI 设计规范集，可以让 Claude Code、Codex 或其他 AI 工具调用，快速生成界面效果预览图。  
参考：`web-design-engineer/`

## 4. 案例：个人知识库搭建

> 用 Codex Cloud + GitHub Actions + Obsidian 搭建一套自动化知识沉淀流程。

### 痛点

1. 看到好文章想总结，但记完就忘在哪了
2. 软件安装/环境配置教程，下次又要重新搜
3. 与 ChatGPT 的有价值对话，没有地方持久保存

### 工具链

| 工具 | 角色 |
|---|---|
| Codex Cloud | Agent 执行端，负责总结和写入 |
| GitHub Actions | 自动化流水线，负责 PR 合并 |
| Obsidian | 本地知识库，负责浏览和搜索 |

### 流程

```
用户输入：总结文章 <url>
    ↓
Agent 抓取并总结文章内容
    ↓
写入 Markdown 文件，提交 PR
    ↓
GitHub Actions 自动合并
    ↓
Obsidian 同步，随时查阅
```

详细搭建步骤见：`Codex云端个人知识库搭建指南.md`

## 5. 总结：AI 编程的核心是写文档

经过一段时间实践，我的核心感受是：

> **AI 编程的上限，取决于你写文档的能力。**

具体来说：

- **定制 Agent 行为** → 写 `AGENT.md`
- **扩展 Agent 技能** → 写 Skill 文件（`skills/*.md`）
- **明确任务边界** → 写 `TODO.md`

不擅长写文档没关系——**让 AI 帮你写 MD 文档**，然后你来审查和修改。  
这本身也是一种 AI 编程实践。
